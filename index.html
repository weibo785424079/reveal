* React Hook是什么 * 官方：Hook 是 React 16.8 的新增特性。 它可以让你在不编写 class 的情况下使用 state 以及其他的 React 特性 * 解释: 即不使用class即可实现逻辑状态的复用，类生命周期特性*
* useState * useCallback * useRef * useMemo * useImperativeHandle * useEffect * useLayoutEffect * useReducer * useContext * useDebugValue
类组件的不足

1、状态逻辑难以复用，需要使用render props 或者 HOC，造成组件嵌套过程，难以理解阅读

2、复杂度高，逻辑分散 componentDidMount绑定事件，componentWillUnmount解除绑定

3、this指向问题

4、代码量较大

(三) 原理浅析 Fiber&Hook

提出两个问题

1、为什么要在函数组件内使用hook

2、为什么不能在条件语句中使用hook

react大致原理的理解程度 这边要和被分享人同步一下理解程度，不然很容易讲不清楚听不懂 首先要介绍fiber原理 ReactElement 然后引入hook初始化，更新过程 完整串联起来，说明组件hook的执行过程 常规的两个问题 为什么要在组件内使用hook 为什么不能再条件中使用hook

总结

优点：简洁的api，高复用，代码量少

缺点：hook拆分过细也有一定的业务理解成本

重点：理解react的运行机制，fiber，render，commitl流程尤为重要
